4 第４回（５月１１日）：関数を書く

4.1 関数とその引数

4.1.1 関数とは？

Rの重要な機能の一つに関数があります。関数は、特定のタスクを実行する「自己完結型」のコードモジュールです。関数は通常、何らかのデータ構造（値、ベクトル、データフレームなど）を受け取り、それを処理して結果を返します。

関数の一般的な表記方法は、関数名の後に括弧を付けるというものです。

function_name(input)

入力は引数と呼ばれ、以下のようなものが含まれます。

  1. 関数がタスクを実行する対象となる物理オブジェクト（任意のデータ構造）
  2. 関数の動作方法を変更する仕様（例：オプション）

すべての関数が引数を取るわけではありません。例えば、

getwd()
## [1] "C:/Users/zhouy/Documents/KU2026Computer/KU2026-1H609"

ただし、ほとんどの関数は複数の引数を取ることができます。関数を呼び出す際に必須引数を指定しない場合、エラーが発生するか、関数はデフォルト値を使用するようにフォールバックします。

デフォルト値は、関数作成者が「標準的なケースでは十分」と判断した標準値です。例えば、グラフで使用するシンボルなどがこれに該当します。ただし、特定の値を使用したい場合は、引数を任意の値に変更してください。

4.1.2 基本機能

これまでのレッスンでは、基本的な関数の例をいくつか使用しました。例えばgetwd()、c()などです factor()。 これらの関数はRの組み込み機能の一部として利用可能であり、以下ではこれらの基本関数をさらにいくつか探っていきます。

外部パッケージやライブラリから関数を取得したり、独自の関数を作成したりすることもできます。これら両方については、後ほど詳しく見ていきましょう。

c()以前、データをベクトルに結合するために使用した関数をもう一度見てみましょう。 この関数が受け取る引数は、数値、文字、または文字列の集合（カンマで区切る）です。 このc()関数は、数値または文字を結合して単一のベクトルを作成する処理を実行します。 また、この関数を使用して、既存のベクトルに要素を追加することもできます。

glengths <- c(10)
glengths <- c(glengths, 90) # adding at the end 
glengths <- c(30, glengths) # adding at the beginning
glengths
## [1] 30 10 90

ここで行われているのは、元のベクトルglengths（3つの要素を含む）の両端にそれぞれ別の要素を追加することです。これを何度も繰り返すことで、ベクトルやデータセットを作成できます。

Rは統計計算に使用されるため、多くの基本関数には数学演算が含まれています。 例えば、関数 が挙げられますsqrt()。 入力/引数は数値でなければならず、出力はその数値の平方根です。 81の平方根を求めてみましょう。

sqrt(81)
## [1] 9

では、単一の値ではなく値のベクトルに対して関数を呼び出した場合（例えば、関数を実行した場合）はどうなるでしょうか？

sqrt(glengths)
## [1] 5.477226 3.162278 9.486833

この場合、タスクはベクトルの各個々の値に対して実行されglengths、それぞれの結果が表示されました。

別の関数を試してみましょう。今回は、オプション（関数の動作を変更する引数）の一部を変更できる関数を使用します。例えば、次のようになりますround。

round(3.14159)
## [1] 3

結果を見ると、 となります3。これは、デフォルトでは最も近い整数に丸められるためです。有効数字の桁数を変えたい場合はどうすればよいでしょうか？

4.2 制御フロー

制御フローには、主に2つのツールがあります。それは、選択肢とループです。選択肢は、ifステートメントやswitch()呼び出しのように、入力に応じて異なるコードを実行できるようにします。ループは、やのようにfor、while通常はオプションを変更しながらコードを繰り返し実行できるようにします。

4.2.1 選択肢（if else）

Rにおけるif文の基本形式は以下のとおりです。

if (condition) true_action
if (condition) true_action else false_action

conditionが TRUEの場合、true_actionが評価されます。conditionが FALSEの場合、オプションfalse_actionが評価されます。

通常、アクションは以下に含まれる複合ステートメントです。

grade <- function(x) {
  if (x > 90) {
    "A"
  } else if (x > 80) {
    "B"
  } else if (x > 50) {
    "C"
  } else {
    "F"
  }
}

if結果を割り当てることができる値を返します。

x1 <- if (TRUE) 1 else 2
x2 <- if (FALSE) 1 else 2

c(x1, x2)
## [1] 1 2

（式全体が1行に収まる場合にのみ、ステートメントの結果を代入することをお勧めしますif。そうでない場合は、読みにくくなる傾向があります。）

else 文を使用せずに単一引数形式を使用すると、条件が NULL の場合、ifは目に見えない形で返します。

greet <- function(name, birthday = FALSE) {
  paste0(
    "Hi ", name,
    if (birthday) " and HAPPY BIRTHDAY"
  )
}
greet("Maria", FALSE)
## [1] "Hi Maria"
greet("Jaime", TRUE)
## [1] "Hi Jaime and HAPPY BIRTHDAY"

4.2.2 ループ (loop)

forループは、ベクトル内の要素を反復処理するために使用されます。ループの基本的な形式は次のとおりです。

for (item in vector) perform_action

vectorの各項目に対して、itemが一度呼び出され、その都度perform_actionの値が更新されます。

for (i in 1:3) {
  print(i)
}
## [1] 1
## [1] 2
## [1] 3

（インデックスのベクトルを反復処理する場合、i、j、k などの非常に短い変数名を使用するのが一般的です。）

注：現在の環境にfor値を割り当てitem、同じ名前の既存の変数を上書きします。

i <- 100
for (i in 1:3) {}
i
## [1] 3

ループを早期に終了させる方法は2つありますfor。

  • next 現在のイテレーションを終了します。
  • break ループ全体から抜け出す for。
for (i in 1:10) {
  if (i < 3) 
    next

  print(i)
  
  if (i >= 5)
    break
}
## [1] 3
## [1] 4
## [1] 5

4.2.2.1 関連ツール

forループは、反復処理したい値のセットが事前にわかっている場合に便利です。わからない場合は、より柔軟な仕様を持つ関連ツールが2つあります。

  • while(condition) action: 実行action中conditionはTRUEです。

  • repeat(action):action永遠に繰り返されます (つまり、 に遭遇するまでbreak)。

Rには、他の言語にあるような構文に相当するものはありませんdo {action} while (condition)。

forループを書き換えて を使用することもできますwhileし、whileループを書き換えて を使用することもできますrepeatが、その逆は成り立ちません。つまり、whileは よりも柔軟性が高くfor、repeatは よりも柔軟性が高いということです。ただし、問題に対しては最も柔軟性の低い解決策を用いるのが良い習慣なので、可能な限りwhileを使用するようにしてください。

4.3 ユーザー定義関数

Rの大きな強みの一つは、ユーザーが関数を追加できることです。時には、小さなタスク（または一連のタスク）を何度も繰り返さなければならない場合があります。このような状況では、独自のカスタム関数を作成すると便利です。関数の構造は以下のとおりです。

name_of_function <- function(argument1, argument2) {
    statements or code that does something
    return(something)
}
  • まず、関数に名前を付けます。
  • そして、それに値を割り当てます。その値とは関数のことです。

関数を定義する際には、必要な引数（関数の動作を変更するための入力値やオプション）のリストを指定し、中括弧で囲んで、それらの引数に対して実行されるタスクを記述します。引数は、スカラー、行列、データフレーム、ベクトル、論理値など、どのような型のオブジェクトでも構いません。引数が何であるかを明示的に定義する必要はありません。

最後に、関数からオブジェクトの値を「返す」ことができます。つまり、その値をグローバル環境に渡すということです。関数の重要な点は、関数内で作成されたオブジェクトは関数の環境にローカルであり、関数の外には存在しないということです。

簡単な関数を作成してみましょう。この関数は数値を入力として受け取り、その二乗値を返します。

square_it <- function(x) {
    square <- x * x
    return(square)
}

これで、他の関数と同じようにこの関数を使用できます。関数の名前を入力し、括弧の中に数値を指定しますx。

square_it(5)
## [1] 25

とても簡単ですよね？この例では、実行したコードは1行だけでしたが、理論的には、ユーザーに「返したい」最終結果を得るために、何行ものコードを使うことも可能です。関数の作成に関しては、ここではほんの表面をなぞったにすぎません！これについては後のレッスンで改めて取り上げますが、興味があれば、この例を参考にしたR-bloggersのサイト でより詳しい情報を見つけることができます。

4.4 パッケージとライブラリ（package）

パッケージとは、特定の機能を追加するために作成された、Rの関数、データ、コンパイル済みコードを明確な形式でまとめたものです。ユーザー提供のパッケージは1万以上あり、現在も増え続けています。

Rのソースコードの一部とみなされ、Rのインストール時に自動的に利用できる標準（または基本）パッケージのセットがあります。基本パッケージには、Rが動作するために必要な基本関数が含まれており、データセットに対して標準的な統計関数やグラフ関数を実行できます。例えば、これまで例で使用してきたすべての関数がこれに該当します。

R においてパッケージが格納されているディレクトリはライブラリと呼ばれます。パッケージとライブラリという用語は同義語として使われることがあり、コミュニティ内でこの問題を解決するために議論が行われてきました。関数を使ってパッケージをロードするのはやや直感に反するlibrary()ため、混乱が生じる可能性があることはお分かりいただけるでしょう。
