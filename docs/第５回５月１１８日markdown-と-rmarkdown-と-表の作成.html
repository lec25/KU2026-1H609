5 第５回（５月１１８日）：Markdown と Rmarkdown と 表の作成

5.1 Markdownとは

Markdown（マークダウン）は文章を記述するための記法（マークアップ言語）の一つです。

Markdown（マークダウン）とは、メールを記述する時のように書きやすくて読みやすいプレーンテキストをある程度見栄えのするHTML文書へ変換できるフォーマットとしてジョン・グルーバーによって開発されました。

以下の特徴があります。

  • 簡単で覚えやすい記述
  • 文章の構造を明示できる
  • Markdownそのままでも理解できる
  • 対応アプリを使うことでより快適に読み書きできる
  • 拡張子は「.md」

Markdownで記述した文章はただのテキストなのでパーサ※によってHTMLに変換します。 このとき、利用するパーサによって出力されるHTMLは若干異なります。 また、利用しているcssによって見栄えも変わります。

※注釈 パーサとは構造化された文書を解析し、別の構文に変換するプログラム

こちらを参照してください：https://qiita.com/Hase-pro/items/16379a0c83f2725e3a11

5.3 表の作成

記述統計や推定結果を示す際には、図表が用いられることが多くあります。図については、ggplot2が最も広く利用されているパッケージです（Base Rも依然として高い人気があります）。一方、表の作成については、現時点では主に二つの選択肢があります。一つは、長年利用されてきたknitrパッケージのkable()（=kbl()）とkableExtraパッケージを組み合わせる方法、もう一つは近年注目を集めているgtパッケージです。

gtは“Grammar of Tables”の略であり、“Grammar of Graphics”を具現化したggplot2の考え方に影響を受けています。つまり、一つの表を複数の要素に分解し、それぞれのレイヤーを積み重ねながら表を構築していくイメージです。ggplot2の考え方や使い方に慣れている読者であれば、gtにも比較的容易に慣れることができるでしょう。

ただし、gtは現在も開発途上にあり、PDF出力との相性は、現時点（2025年1月15日；gt version 0.11.1）では必ずしも十分とは言えません。そのため、表をPDF形式で出力する場合には、knitrのkable()とkableExtraを組み合わせる方法を推奨します。

また、以前はgt単体の機能はkable() + kableExtraと比較して限定的でしたが、近年ではgtExtrasの登場により、多くの機能を同様に実現できるようになっています。

gtパッケージはいずれもHTML出力とLaTeX出力の両方に対応していますが、ここではHTML出力を中心に紹介します。LaTeX出力についても、引数を追加することで比較的容易に利用できるため、詳細は各パッケージの公式ドキュメントやヴィネット等を参照してください。
